# BRD - Spring Boot Query Debugger Library

## Product Name

**SQL Insight**

Tagline:  
**Developer Observability for JPA & Spring Boot**

---

# 1. Objective

Build an **open-source Spring Boot library** that provides real-time insights into database queries executed through Hibernate/JPA.

The library should help developers:

- Debug SQL queries
    
- Detect performance issues
    
- Identify N+1 queries
    
- Track slow queries
    
- Understand query sources (repository/service)
    
- Visualize query metrics through a **management dashboard**
    

The system must be **safe for production environments** by allowing the dashboard to be disabled through configuration.

---

# 2. Problem Statement

Spring Boot developers often face the following issues:

• SQL queries generated by Hibernate are difficult to debug  
• Query logs are messy and hard to interpret  
• N+1 query issues go unnoticed  
• Performance problems occur without clear visibility  
• Existing tools like p6spy lack modern dashboards

Developers need a **lightweight observability tool specifically for Spring Boot applications**.

# 5. Key Features

## Query Monitoring

Capture:

• SQL query text  
• Execution time  
• Query source (method/class)  
• Execution timestamp  
• Query parameters

---

## Slow Query Detection

Detect queries exceeding a configurable threshold.

Example:

```yaml
sql-insight:
  slow-query-threshold: 200
```

This flags queries slower than **200 milliseconds**.

---

## N+1 Query Detection

Detect repeated execution of similar queries within a short timeframe.

Example:

```
SELECT * FROM orders WHERE user_id = ?
```

If executed repeatedly inside a loop, flag as potential N+1 issue.

---

## Query Source Mapping

Each query should identify:

- Controller
    
- Service
    
- Repository
    

Example output:

```
Query: SELECT * FROM users WHERE id = ?

Triggered by:
UserRepository.findById()

Called from:
UserService.getUser()
```

# 6. Annotation System

The library must provide custom annotations to allow developers to gain additional insights.

## @QueryTrace

Tracks query activity inside a method.

Example:

```java
@QueryTrace
public List<User> getUsers() {
    return userRepository.findAll();
}
```

This annotation will:

• Track queries executed inside the method  
• Measure execution time  
• Display query statistics in dashboard

---

## @SlowQueryAlert

Raises alert if a query exceeds threshold.

Example:

```java
@SlowQueryAlert(threshold = 300)
public List<Order> getOrders() {
    return orderRepository.findAll();
}
```

---

## @DisableQueryTracking

Disable monitoring for sensitive operations.

Example:

```java
@DisableQueryTracking
public void updatePassword() {
    ...
}
```

---

## @QueryLabel

Allows tagging of queries.

Example:

```java
@QueryLabel("User Fetch")
public User findUser(Long id)
```

Labels will appear in dashboard analytics.

# 7. System Architecture

Core Architecture Components:

### Query Interceptor

Intercepts SQL statements using Hibernate.

Primary approach:

```
Hibernate StatementInspector
```

Responsibilities:

• Capture SQL query  
• Start execution timer  
• Capture context information

---

### Query Collector

Stores executed queries in memory.

Recommended structure:

```
ConcurrentLinkedQueue
```

Limits:

```
maxQueries = 1000
```

Older queries are automatically removed.

---

### Query Analyzer

Processes collected queries to generate insights.

Calculations include:

• Average execution time  
• Slow queries  
• Query frequency  
• N+1 detection

---

### Metrics Engine

Aggregates query statistics for dashboard.

Metrics include:

• Total queries executed  
• Queries per second  
• Average query time  
• Top slow queries

---

### Dashboard API

Backend endpoints used by UI.

Endpoints:

```
GET /sql-insight/metrics
GET /sql-insight/queries
GET /sql-insight/slow-queries
GET /sql-insight/nplus1
GET /sql-insight/health
```

# 8. Configuration

Configuration must be provided via Spring Boot properties.

Example configuration:

```yaml
sql-insight:
  enabled: true
  dashboard-enabled: true
  slow-query-threshold: 200
  max-queries: 1000
  detect-nplus1: true
```

---

Production configuration example:

```yaml
sql-insight:
  dashboard-enabled: false
```

This completely disables dashboard endpoints.
